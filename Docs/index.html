Introduction

A while back I put the source code for two small plugins, niftyperforce and niftysolution, up on google code for everybody to download and use. The plugins are a collection of handy utilities that you can find to boost your productivity if you're using visual studio 2005 and perforce. This documentation will go through some of the features.

NiftySolution

NiftySolution has three features, open any file in the solution fast through a small dialog, toggle header/.cpp file quickly and last a timer for how long it took to build the complete solution. The plugin will register a couple of commands which you can either access through the interface ctrl-alt-a or you can bind them to a keyboard shortcut.

Fig 1: Opens a file in the solution dialog. You can access this through the command Aurora.NiftySolution.Connect.NiftyOpen.
Fig 2: Shows the automatic reporting of the total solution time when building.

NiftyPerforce

Niftyperforce is a small lightweight interface to the revision control system perforce. It registers a couple of context menus in the solution explorer as well as a couple of event handlers and commands. The event handlers are automatic reaction to add/delete items in the solution (such as files or projects). These events will trigger perforce add/remove operations. There is a also an option to automatically have perforce checkout any file you start to edit inside visual studio. Bring up the options with the command Aurora.NiftyPerforce.Connect.Configuration.

Fig 3: Bring up the config dialog with Aurora.NiftyPerforce.Connect.Configuration.
Fig 4: Shows that there is a context menu "p4 edit" which works for multiselect items in the solution explorer.
Fig 5: You can also rename files in the solution explorer and it will call the correct functions (move the file, update the visual studio project, update perforce).
Fig 6: Here is a list of the commands that are registered for the plugin. Notice that there is a edit all modified command, which is very useful after a huge search and replace operation with "keep modified files open". Now you can call this command and have visual studio automatically check everything out.

On of the prerequsite of the plugin is that you have p4.exe in your path and some way for p4 to find the clientspec and server. Either if it's the default config or through the P4CONFIG environment variable.
